# The Operator Playbook

Five workflows that actually run my week. Each is opinionated, mechanical, and battle-tested through 7 products in 4 months.

| Workflow | When to read it |
|----------|-----------------|
| [Ship a product in a day](ship-a-product-in-a-day.md) | About to start a new product. Want a defensible cadence. |
| [Parallel projects](parallel-projects.md) | More than 2 products in flight. Context switching is killing you. |
| [Obsidian as context](obsidian-as-context.md) | Memory across sessions matters. Sessions cost more than they should. |
| [Content pipeline](content-pipeline.md) | You ship content alongside product. You don't want it to eat the week. |
| [Solo ops](solo-ops.md) | You handle support + billing + infra alone. Looking for the muscle. |

## Read order

If you're new to the stack and trying to copy the operating model, read in this order:

1. **parallel-projects.md** — the meta-discipline. Without this, the others are local optimizations.
2. **obsidian-as-context.md** — the loop that makes sequential focus actually work.
3. **ship-a-product-in-a-day.md** — the day-shape that produces shippable evidence.
4. **solo-ops.md** — what you do when the products start producing real-world load.
5. **content-pipeline.md** — only when you're ready to ship a brand alongside the product.

## What these are not

These are not strict prescriptions. They are the workflows that compose well with the [stack](../stack/), and that I personally run.

Yours will diverge — different products, different brand, different ops surface. The point of writing them down is to have something concrete to disagree with, fork from, and improve.

## Common pattern across all five

Every workflow has the same shape:

1. **A clear before-state** — what triggers running it
2. **Mechanical steps** — not "be thoughtful," but "do step 1, then step 2"
3. **Anti-patterns** — what I learned the hard way
4. **An end-state** — what's true after the workflow runs

If your version of these workflows can't pass the "could a teammate execute this on Tuesday without me explaining" test, they're not ready. Make them more mechanical.
