Operator-to-operator. Six tweets in the body plus a first reply with the link. No emoji, no "thrilled to announce", no "huge if true."

Each tweet has been counted. All ≤270 chars. The link reply sits below the pinned tweet so the hook keeps the timeline real estate.

## Three hook variants (pick one)

The hook tweet is what the timeline scrolls past or stops at. It does the work for the rest of the thread. Three different angles on the same story so you can pick the one that fits your mood that morning.

### Variant A — the metric hook

> 7 products in 4 months. Solo. First line of code I ever wrote was January.
>
> The stack and the playbook are now public. It is mostly other people's work, glued by my install path and the workflows I run from. Thread on what is in it and what is not.

(269 chars.)

### Variant B — the anti-engineer hook

> I am not an engineer. I started writing code in January with Cursor and Claude.
>
> Four months later I run 3 live sites, 4 SaaS codebases, a YouTube pipeline, a daily-use AI assistant, and one stack that holds it together. Open-sourcing the stack today.

(259 chars.)

### Variant C — the geography hook

> I run 7 AI products from Buenos Aires. Solo. No CS background.
>
> The setup is not magic. It is Claude Code, four marketplaces, six hooks, twelve cookbook recipes, and a loop that compounds. Pushing the whole stack public now so other operators can stop reinventing it.

(270 chars.)

**Recommended default: Variant B.** It does the most work in the fewest lines: states the operator framing, names the surface area, lands the launch in one sentence. Variant A is a touch number-heavy and reads like a humblebrag without the rest of the context. Variant C anchors on geography, which is interesting but less universal.

## Notes for the operator

- **When to post:** weekday, 9-11 AM ET is the conventional best window for tech-X (best guess, not measured for your audience). If your audience is LATAM-heavy, 10 AM-12 PM Buenos Aires (1-3 PM ET) is also viable.
- **Pin the hook for 48h.** After that, unpin and let it slide.
- **Reply to the first 30 mentions personally.** First-hour replies set the tone of the thread.
- **Do not quote-tweet yourself in the first 6 hours.** Let the original thread breathe.
- **If a tweet underperforms in the first 30 minutes, do not delete-and-repost.** That kills the thread's reach. Take the L and learn from it.
- **One follow-up tweet in 24h** with one concrete number from the launch (stars, issues, PRs) keeps the thread alive without looking thirsty.
- **Do not** add hashtags. They read as desperate on this kind of post.
